## 安装与运行

### Q1：支持哪些操作系统？

支持 Linux、macOS 和 Windows 10 及以上。macOS 同时提供 Intel 和 Apple Silicon (arm64) 版本。

### Q2：为什么首次启动时控制台会有一行 SqliteError？

首次启动时，应用会在全新的数据目录下创建数据库。该日志属于数据库初始化的正常流程，不影响使用，后续启动不会再出现。

### Q3：macOS 提示"无法验证开发者"或"已损坏"怎么办？

珊瑚音乐为开源项目，未付费购买 Apple 开发者签名。请在终端执行以下命令放行：

```sh
sudo xattr -dr com.apple.quarantine /Applications/Coral\ Music.app
```

### Q4：Windows 启动时被杀毒软件拦截？

由于本项目未购买代码签名证书，部分杀毒软件可能误报。你可以将应用添加到信任列表，或从源码自行构建。

## 音源与播放

### Q1：为什么无法播放在线音乐？

珊瑚音乐本身不内置在线音源接口，在线音频数据来自软件设置中的"自定义源"。请按以下步骤检查：

1. 进入 **设置 → 自定义源**，确认已导入至少一个可用的音源脚本
2. 确认该音源已"设为当前"
3. 确认音源状态为正常（无报错提示）

### Q2：如何添加自定义音源？

有两种方式：

- **文件导入**：设置页点击"添加音源"，选择本地的 `.js` 音源脚本文件
- **在线导入**：设置页点击"在线导入"，输入音源脚本的 URL

导入后会自动设为当前音源。

### Q3：播放时提示"当前没有可用音源"怎么办？

表示当前未启用任何自定义音源，或启用的音源返回了空链接。请：

1. 检查是否已导入并启用了音源
2. 尝试切换到其他音源
3. 确认音源脚本是否支持当前歌曲的音源平台

### Q4：播放在线音乐时显示的采样率和比特率是实际值吗？

是的。珊瑚音乐会通过 Range 请求获取音频文件的前 64KB 数据，解析文件头（MP3/FLAC/WAV）获取实际的采样率和比特率。探测是异步进行的，播放开始时可能先显示理论音质规格，探测完成后自动更新为实际值。

### Q5：切换音质后播放没有变化？

切换音质后会自动刷新当前歌曲的播放链接。如果切换后仍然播放之前的音质，可能是：

- 当前音源不支持所选音质，已自动降级
- 网络请求失败，请重试

### Q6：支持哪些在线音质？

| 音质标识           | 说明                  |
| ------------------ | --------------------- |
| `128k`             | 有损压缩，128 kbps    |
| `192k`             | 有损压缩，192 kbps    |
| `320k`             | 有损压缩，320 kbps    |
| `FLAC`             | 无损压缩，44.1 kHz    |
| `FLAC Hi-Res`      | 无损压缩，96 kHz      |
| `Hi-Res`           | 高解析度无损，192 kHz |
| `Master`           | 高品质无损            |
| `Atmos` / `Atmos+` | 空间音频              |

实际可用音质取决于当前音源和歌曲本身的支持情况。

## 本地音乐

### Q1：支持哪些本地音频格式？

| 格式                         | 播放方式              |
| ---------------------------- | --------------------- |
| MP3                          | 内置支持              |
| FLAC                         | 内置 WASM 解码器      |
| WAV                          | 内置支持              |
| M4A / AAC                    | 内置支持              |
| OGG / Opus                   | 内置支持              |
| DSD / DFF / DSF / ISO / SACD | 需外部解码器 (FFmpeg) |

### Q2：如何启用外部解码器播放 DSD 等格式？

1. 安装 FFmpeg 并确保可执行
2. 进入 **设置 → 本地解码**
3. 启用外部解码器
4. 选择解码器路径（或填写 `ffmpeg`）
5. 添加需要的扩展名（如 `dsf`、`dff`、`iso`）

### Q3：本地音乐的歌词从哪里来？

本地音乐支持以下歌词来源（按优先级）：

1. 音频文件内嵌歌词
2. 本地同名 `.lrc` 文件
3. 在线匹配（基于歌曲名和歌手名搜索）

## 下载

### Q1：下载的歌曲保存在哪里？

默认保存在桌面。可在 **设置 → 下载设置 → 保存位置** 中修改。

### Q2：下载失败提示"下载地址失效"怎么办？

音频链接可能已过期。请在下载列表中右键点击失败的任务，选择"重试"或"刷新链接"，应用会重新获取下载链接。

### Q3：下载时可以同时保存歌词吗？

可以。在 **设置 → 下载设置** 中开启"下载歌词"。支持以下歌词格式：

- 标准歌词 (`.lrc`)
- 逐字歌词 (LRC Plus)
- 翻译歌词
- 罗马音歌词

## 同步

### Q1：同步功能有什么用？

同步功能允许你在多台设备之间保持歌单和黑名单一致。支持服务端模式和客户端模式：

- **服务端模式**：本机作为同步服务端，其他设备连接到本机
- **客户端模式**：本机作为客户端，连接到其他设备上的同步服务端

### Q2：同步时如何处理歌单冲突？

同步提供以下模式：

- **合并**：将两端歌单合并，保留各自的歌曲
- **覆盖**：用一端的歌单覆盖另一端
- **完全覆盖**：不做合并，直接用一端的数据替换另一端

合并模式会根据歌曲 ID 去重，不会产生重复歌曲。

### Q3：同步数据存储在哪里？

同步数据存储在本机的 `userData/sync/` 目录下，按用户名分隔。不同用户之间的数据相互隔离。

## 数据与存储

### Q1：应用数据存储在哪个目录？

| 平台    | 路径                                                         |
| ------- | ------------------------------------------------------------ |
| macOS   | `~/Library/Application Support/coral-music-desktop/CoralDatas/` |
| Windows | `%APPDATA%/coral-music-desktop/CoralDatas/`                     |
| Linux   | `~/.config/coral-music-desktop/CoralDatas/`                     |

### Q2：珊瑚音乐的数据会与原版 lx-music-desktop 冲突吗？

不会。珊瑚音乐强制使用独立的 `coral-music-desktop` 数据目录，与原版 `lx-music-desktop` 完全隔离，互不影响。

### Q3：如何迁移原版 lx-music-desktop 的数据？

珊瑚音乐首次启动时会自动尝试迁移旧版数据（歌单、歌词、快捷键等）。若自动迁移失败，可手动将原版数据复制到珊瑚音乐的 `CoralDatas/` 目录。

### Q4：如何备份应用数据？

备份 `CoralDatas/` 目录即可。主要数据文件：

| 文件　　　　　　　　  | 内容　　　　　　　　　　　　　　　　　 |
| --------------------- | -------------------------------------- |
| `coral.data.db`　　　　  | 歌单、歌曲信息、歌词（SQLite 数据库）  |
| `config_v2.json`　　  | 应用设置　　　　　　　　　　　　　　　 |
| `user_api.json`　　　 | 自定义音源脚本　　　　　　　　　　　　 |
| `hot_key.json`　　　  | 快捷键配置　　　　　　　　　　　　　　 |
| `lyrics_edited.json`  | 编辑过的歌词　　　　　　　　　　　　　 |

### Q5：重置应用数据后音源不见了怎么办？

自定义音源数据存储在 `user_api.json` 中。重置数据后需要重新导入音源脚本。建议提前备份 `user_api.json`。

## 开发与构建

### Q1：如何本地开发？

```sh
npm install
npm run dev
```

### Q2：如何构建生产版本？

```sh
npm run build
```

### Q3：如何打包分发？

```sh
npm run pack:dir   # 仅打包目录，不安装
npm run pack       # 完整打包（dmg/exe/deb 等）
```

### Q4：如何运行验证测试？

```sh
npm run smoke:full  # 完整无网络验证（构建+lint+类型检查）
npm run smoke:download  # 下载功能烟雾测试
```

### Q5：开发时 Vite 报 504 (Outdated Optimize Dep) 怎么办？

删除依赖缓存后重启开发服务器：

```sh
rm -rf node_modules/.vite
npm run dev
```
